# Honeycomb UX Guidelines

## 1. 表单 UX

### 1.1 输入反馈
- 实时校验：失焦（onBlur）时触发，不在输入过程中报错
- 校验通过后清除错误提示
- 提交时统一校验所有字段，第一个错误字段自动聚焦

### 1.2 占位符与默认值
- 占位符颜色 `var(--color-text-2)`，描述期望格式，如「请输入手机号」
- 有默认值的字段预填，不用占位符代替
- 编辑表单预填已有值，新建表单不预填（日期等除外）

### 1.3 表单分组顺序
- 基本信息 → 详细信息 → 关联信息 → 备注
- 单组字段不超过 12 个，超出拆分多 Card

### 1.4 下拉选项
- ≤5 个选项：Radio Group；>5 个：Select
- >20 个选项：Select 开启 `showSearch`
- 级联选择：TreeSelect 或多级 Select

## 2. 反馈与提示 UX

### 2.1 Toast（轻提示）
- 成功操作（保存/删除/导出）→ success Toast，3秒自动消失
- 网络异常 → error Toast，5秒消失
- 位置：页面右上角，不遮挡主内容
- 同时最多显示 3 条，新的顶部插入

### 2.2 Modal 确认
- 危险操作（删除/停用/清空）必须 Modal 二次确认
- 确认文案：具体说明后果，避免「确认操作？」这类模糊文案
- 好的示例：「删除后数据不可恢复，确认删除「XX房源」吗？」
- 确认按钮：danger 样式；取消按钮：default 样式

### 2.3 空状态
- 有数据时才显示表格，无数据显示空状态组件
- 空状态包含：图标 + 主文案 + 引导操作按钮
- 主文案：「暂无{业务名称}」
- 搜索无结果：「未找到符合条件的结果，请调整筛选条件」+ 重置按钮

### 2.4 加载状态
- 列表初始加载：骨架屏（Skeleton）优于 Spinner
- 按钮操作中：Button `loading:true`，防重复点击
- 加载超 300ms 才显示 loading，避免闪烁

## 3. 导航 UX

### 3.1 面包屑
- 表单页和详情页必须有面包屑
- 格式：「模块名 / 列表页名 / 当前页名」
- 最后一项不可点击（当前页），前面项可点击跳转

### 3.2 侧边导航
- 当前激活菜单高亮：背景 `var(--color-primary-light)`，文字 `var(--color-primary)`
- 二级菜单默认收起，激活时展开
- 支持折叠（collapsed），折叠时只显示图标
- 导航分组用小标题分隔，颜色 `var(--color-text-2)`

### 3.3 Tab 导航
- 详情页/设置页使用 Tab 分组内容
- Tab 数量不超过 8 个，超出用下拉「更多」
- 切换 Tab 保留当前 Tab 的滚动位置

## 4. 数据展示 UX

### 4.1 表格列设计
- 重要字段（名称/编号）放前两列，操作列固定右侧
- 状态列必须用 Tag，不用纯文字
- 金额列右对齐，保留两位小数，加货币符号
- 时间列格式统一：`YYYY-MM-DD HH:mm`
- 长文本列设置最大宽度 + `ellipsis`，hover 显示 tooltip

### 4.2 数字与金额
- 大数字（>10000）用「万」为单位，保留1位小数
- 金额格式：`¥ 1,234.56`（千分位分隔）
- 统计卡片的数字使用 `tokenRef("typography.display-md")`
- 同比/环比变化：上升用 `var(--color-success)`，下降用 `var(--color-error)`

### 4.3 状态设计
| 业务状态 | Tag 类型 | 说明 |
|---|---|---|
| 正常/有效/已完成 | success | 绿色 |
| 待处理/进行中/待审批 | info/warning | 蓝色/黄色 |
| 异常/已逾期/已拒绝 | error | 红色 |
| 已停用/已关闭/草稿 | default | 灰色 |

### 4.4 时间显示
- 24小时内：「X小时前」或「X分钟前」
- 超过24小时：显示完整日期时间
- 创建时间/更新时间等辅助信息用 `var(--color-text-2)` 展示

## 5. 操作反馈时序

| 操作 | 反馈顺序 |
|---|---|
| 保存表单 | 按钮 loading → 接口成功 → Toast成功 → 跳转列表页 |
| 删除记录 | Modal确认 → 确认按钮loading → 接口成功 → Toast成功 → 刷新列表 |
| 导出数据 | 按钮loading → 接口成功 → Toast「导出成功，请查看下载」→ 按钮恢复 |
| 表单校验失败 | 字段下方显示错误提示 → 第一个错误字段聚焦 → 按钮不进入loading |
| 接口请求失败 | 按钮恢复 → Toast error提示 → 不跳转页面 |
